# FossilSense 1.4.0 Phase 4 name-index compaction benchmark

status: current

measured_at: 2026-07-12

database: schema-15 Wine full index (local, generated)

## Scope

This measurement validates background compaction for the segmented name index. Dirty publication first atomically publishes the changed-file delta and releases the publisher. If the table has at least 64 delta segments, or retained delta slots exceed 25% of base entries, a blocking worker builds a new compact base. It checks the captured `EngineEpoch` both before and after construction, so a newer dirty/full/project-context publication causes the obsolete result to be discarded.

The benchmark preserves the first five dirty replacements as the latency/memory measurement, then repeatedly replaces the worst-case Wine file until the entry threshold is crossed. Nine segments of 14,173 rows cross the 25% threshold for the 495,645-row base. Compaction is measured separately; its full-base allocation must not be hidden inside dirty publication latency.

## Wine results

Three adjacent release runs:

| Metric | Run 1 | Run 2 | Run 3 | Median / interpretation |
|---|---:|---:|---:|---:|
| Typed-row SQL load | 437 ms | 435 ms | 430 ms | 435 ms |
| Cold in-memory build | 681 ms | 709 ms | 707 ms | 707 ms, <800 ms gate |
| Dirty replacement | 10.711 ms | 12.302 ms | 11.238 ms | 11.238 ms, <100 ms gate |
| Dirty Private Bytes delta | 9.989 MiB | 9.984 MiB | 9.984 MiB | 9.984 MiB, <32 MiB gate |
| Compaction input | 9 segments | 9 segments | 9 segments | entry threshold |
| Background compact build | 615 ms | 638 ms | 662 ms | 638 ms |
| Compact-build Private Bytes delta | 97.52 MiB | 97.51 MiB | 98.39 MiB | 97.52 MiB median |

Compaction intentionally rebuilds all active entries while the old immutable table remains alive, so its temporary memory is substantially larger than one delta. The optimization is accepted because this work is outside the dirty response/publisher critical section, bounds retained segments, and does not regress the already-passing dirty latency or memory gates. A future compact arena/streaming builder can reduce the compact-build peak independently.

## Correctness boundary

- Compacting 64 synthetic delta generations preserves active exact/prefix results and removes all delta segments.
- A compaction may publish only if the expected `EngineEpoch` is still current; retrying with the stale epoch leaves the replacement snapshot untouched.
- Compaction preserves `SemanticGeneration` and all non-name snapshot components, allocates a new `EngineEpoch`, and clears completion memos.
- The old immutable snapshot remains usable while the blocking worker constructs the compact replacement.
