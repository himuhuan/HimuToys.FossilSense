# FossilSense 1.4.0 Phase 4 streaming name-index benchmark

status: current

measured_at: 2026-07-12

database: schema-15 Wine full index (local, generated)

## Scope

This measurement validates the second cold-build optimization. Production full publication and CLI symbol queries now visit borrowed SQLite name rows and intern each label/path/lowercase spelling directly into the immutable index. They no longer first materialize 495,645 `NameTableSymbolRow` values with owned label/path/kind strings and then allocate/intern those strings a second time. Project ownership is computed once per unique path during the build.

Dirty publication keeps its narrow owned-row loader because it reads only changed paths. The legacy full typed-row API remains as a test oracle, not a production cold path.

## Wine before / after

The prior segmented build measured SQL typed-row loading and in-memory construction separately. End-to-end is their sum:

| Metric | Prior run 1 | Prior run 2 | Prior run 3 | Prior median |
|---|---:|---:|---:|---:|
| Typed-row SQL load | 437 ms | 435 ms | 430 ms | 435 ms |
| In-memory build | 681 ms | 709 ms | 707 ms | 707 ms |
| End-to-end cold build | 1,118 ms | 1,144 ms | 1,137 ms | 1,137 ms |

Three adjacent release streaming runs:

| Metric | Run 1 | Run 2 | Run 3 | Median / change |
|---|---:|---:|---:|---:|
| SQLite -> immutable NameIndex | 1,024 ms | 992 ms | 1,034 ms | 1,024 ms, -9.9% |
| Dirty replacement | 11.979 ms | 12.833 ms | 13.652 ms | 12.833 ms, <100 ms |
| Dirty Private Bytes delta | 15.39 MiB | 15.40 MiB | 15.39 MiB | 15.39 MiB, <32 MiB |
| Background compact build | 749 ms | 735 ms | 772 ms | 749 ms |
| Compact-build Private Bytes delta | 97.51 MiB | 97.50 MiB | 98.39 MiB | 97.51 MiB |

The optimization is effective and preserves the dirty gates, but the stricter end-to-end cold-build target remains unmet: 1,024 ms is still above 800 ms. Phase 4 therefore continues with compact string/entry IDs; mmap is evaluated only after the compact streaming builder is measured.

## Correctness boundary

- Borrowed values cannot escape the store callback; `rusqlite` remains inside the store boundary.
- Visitor output matches the existing typed-row wrapper for ID, label, source, path, kind, directly-included state, and row order.
- A real indexed workspace produces identical ranked hits and project postings through streaming and typed-row builders.
- Production full snapshot publication, CLI symbol query, and real-index query tests use the streaming path.
