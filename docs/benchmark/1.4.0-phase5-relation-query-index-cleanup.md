# FossilSense 1.4.0 Phase 5 relation query-index cleanup

status: current

measured_at: 2026-07-12

database: schema-15 Wine full index (local, generated)

## Scope

This cleanup removes the last workspace-catalog API surface. The former `RelationCatalog` type is now a request-scoped `RelationQueryIndex`, built only from facts admitted by SQL scan and candidate-expansion budgets. Full-workspace anchor/call-site visitors, schema-14 oracle construction, exact catalog totals/timers, and `usize::MAX` incoming/outgoing materializers were deleted. Tests use explicit bounded pages.

CLI and the benchmark script now report `relation_query_*` counters and use schema-15 `*-rebuild.sqlite` databases. The candidate resolver, evidence, grouping, ordering, cursor/page behavior, and DTO materialization algorithm are unchanged.

## Wine regression check

Release process benchmark after cleanup:

| Case | Result | Phase 1-2 comparison |
|---|---:|---:|
| Medium fan-in first cold query | 168.035 ms | <250 ms cold gate |
| Medium fan-in warmed queries | 18.735 / 20.129 ms | prior 18-19 ms |
| High-frequency query median | 143.587 ms | prior 136-140 ms; within observed run spread |
| High-frequency process elapsed median | 330.443 ms | prior 299-320 ms |
| High-frequency peak Private median | 15.08 MiB | prior 13-15 MiB; far below 128 MiB gate |

The high-frequency request still stops at 8,192 raw sites, returns 200 relations, and builds a bounded query index with 1,447 entities, 16,381 relations, and 16,382 relation-site refs. The small timing difference is not a claimed optimization win; this is a behavior-preserving API cleanup, and the result shows no material regression or loss of budget bounds.

## Correctness boundary

- Production/source search has no `RelationCatalog`, `with_overlays`, `rebuild_relation_catalog`, full call-fact visitor, unlimited relation API, or `catalog_*` metric.
- Lazy store query tests assert the expected resolved caller/callee directly and cover dirty shadow plus cross-document overlay merge without a full-workspace oracle.
- High ambiguity, raw scan, candidate expansion, site, relation page, cursor/revision, and standard-LSP limits remain tested.
- The schema-14 database rejection observed during the first benchmark attempt is expected; the script now selects schema-15 rebuild databases.
