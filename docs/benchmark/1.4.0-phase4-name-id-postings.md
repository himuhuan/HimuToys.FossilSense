# FossilSense 1.4.0 Phase 4 name-ID postings benchmark

status: current

measured_at: 2026-07-12

database: schema-15 Wine full index (local, generated)

## Scope

This measurement validates the final Phase 4 cold-build optimization. The previous compact segment sorted all 495,645 entries with repeated `(lowercase name, original name)` string comparisons. The replacement sorts only 381,286 unique name IDs, counts entries per name ID, computes offsets in sorted-name order, and fills stable entry postings in one linear pass.

Names with identical `(lower, original)` are the same interned name ID, so filling entries in insertion order preserves the previous stable-sort tie behavior. Case variants still use original spelling as the secondary key.

## Wine before / after

The diagnostic now splits SQLite visitation/interning from immutable finalization. Three adjacent release runs:

| Metric | Compact baseline median | Run 1 | Run 2 | Run 3 | ID-posting median / change |
|---|---:|---:|---:|---:|---:|
| SQLite visit + intern | — | 632 ms | 652 ms | 637 ms | 637 ms |
| Arena/posting finalize | 276 ms | 154 ms | 128 ms | 130 ms | 130 ms, -52.9% |

| Dirty Private Bytes delta | 11.69 MiB | 11.43 MiB | 11.43 MiB | 11.12 MiB | 11.43 MiB, <32 MiB |

The end-to-end median is below the 800 ms Phase 4 target. Following the plan's decision gate, FossilSense does not add a custom generation-keyed sidecar or mmap format: the compact streaming builder is already within budget, avoiding a second persistence format and its publication/recovery compatibility burden.

## Rejected A/B

Capacity pre-sizing was tested immediately before this optimization. It ran `COUNT(*)` and reserved the entry/name/path arenas before visiting rows. Three release runs measured 1,096 / 1,157 / 1,133 ms (median 1,133 ms), a 24.9% regression from the 907 ms compact baseline. The extra table scan cost more than the avoided reallocations, so the change was fully reverted and not committed.

## Correctness boundary

- A targeted ordering test covers case variants plus duplicate identical spellings and proves stable insertion order for the latter.
- Existing exact/prefix fast-path, full-scan parity, narrowing, segmentation, tombstone, project-quota, coloring, and compaction suites remain unchanged.
- The diagnostic reports only aggregate counts/timings and never prints names or paths.
