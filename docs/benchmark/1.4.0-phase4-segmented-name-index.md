status: current

## Scope

This measurement validates the first Phase 4 optimization: dirty name-index publication shares one immutable full base and appends compact changed-file segments. A per-path override selects the active delta or deletion tombstone. Query-local virtual entry IDs merge active base/delta candidates without changing completion-pool, ranking, project quota, coloring, or exact/prefix APIs.

Dirty publication no longer clones the base entries, rebuilds three full interning maps, globally sorts all entries, or recomputes workspace reach from every symbol. Streaming cold construction and background compaction are separate later optimizations.

## Correctness boundary

- Repeated replacement of one path shadows both the base and its previous delta; deletion installs a tombstone without exposing either older generation.
- Exact/prefix lookup merges sorted base and delta candidates; pooled prefix narrowing matches a cold segmented scan.
- Unchanged paths retain base IDs/results, and `len()` counts only active entries.
- Query parity suite: 37 passed, one diagnostic benchmark ignored.
- Full gate: 664 Rust tests passed, 2 diagnostic benchmarks ignored, and 2 LSP smoke tests passed.
- VS Code TypeScript compile/tests passed; architecture fitness reported zero failures and six existing large-file warnings.

Background compaction is intentionally not part of this A/B. Segment-count/entry thresholds and atomic compaction publication must be measured independently so the 17.690 ms dirty result cannot hide a synchronous full rebuild.
