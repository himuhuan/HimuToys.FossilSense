# FossilSense 1.4.0 Phase 4 compact name-index benchmark

status: current

measured_at: 2026-07-12

database: schema-15 Wine full index (local, generated)

## Scope

This measurement validates compact segment-local name/path/project arenas. Each published symbol entry now stores a symbol ID, three `u32` arena IDs, kind, and flags. Original/lowercase names, paths, and `ProjectKey` values are stored once per segment. `all_workspace_reach` walks unique path IDs instead of symbol entries.

Cold SQLite visitation writes these compact entries directly. Dirty segments use the same representation. Background compaction and marker-only project retagging consume borrowed compact-entry views, so they do not first reconstruct a workspace-sized vector of owned strings.

## Layout

Wine aggregate counters:

| Counter | Value |
|---|---:|
| Symbol entries | 495,645 |
| Unique names | 381,286 |
| Unique paths | 7,053 |
| Compact entry inline size | 24 bytes |
| Previous transient owned entry inline size | 112 bytes |

This benchmark builds without a project-context fixture, so its project arena is empty. The real-index parity test supplies project context and verifies project postings/results against the typed-row oracle.

## Before / after

Three adjacent release runs after compact-to-compact compaction was enabled:

| Metric | Streaming median | Compact run 1 | Compact run 2 | Compact run 3 | Compact median / change |
|---|---:|---:|---:|---:|---:|
| SQLite -> immutable NameIndex | 1,024 ms | 879 ms | 907 ms | 918 ms | 907 ms, -11.4% |
| Dirty replacement | 12.833 ms | 9.800 ms | 10.445 ms | 10.481 ms | 10.445 ms, -18.6% |
| Dirty Private Bytes delta | 15.39 MiB | 11.69 MiB | 11.82 MiB | 11.59 MiB | 11.69 MiB, -24.0% |
| Background compact build | 749 ms | 554 ms | 508 ms | 551 ms | 551 ms, -26.4% |
| Compact-build Private Bytes delta | 97.51 MiB | 87.86 MiB | 87.71 MiB | 87.65 MiB | 87.71 MiB, -10.1% |

Against the original typed-row + owned-entry end-to-end median of 1,137 ms, streaming plus compact arenas reduce cold construction by 20.2%. The optimization is accepted, but Phase 4 is not complete: the 907 ms median remains above the 800 ms cold target. Capacity pre-sizing and remaining builder costs must be measured before the mmap decision.

## Correctness boundary

- A layout guard fails if `CompactNameEntry` grows beyond 24 bytes and regains per-symbol pointers.
- Query exact/prefix/fuzzy ranking, narrowing pools, project quotas, color-kind counts, tombstones, and repeated path replacement use resolved borrowed entry views and remain unchanged.
- Compaction preserves active results and removes all deltas without materializing owned entries.
- Marker-only retagging recomputes project ownership once per path into the new project arena.
- Full query suite and expected-epoch publication tests cover both compact base and compact delta segments.
