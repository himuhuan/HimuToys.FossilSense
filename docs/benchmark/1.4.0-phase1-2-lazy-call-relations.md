## High-frequency budget behavior

The high-frequency case has 27,672 persisted raw sites. The lazy query scans at most 8,192 rows, returns a 200-relation page, marks `scan_limited`, and does not calculate or serialize an exact workspace total. Across the three runs:

- peak Private Bytes: 13.3–15.2 MiB;
- service query: 136–140 ms;
- request-local call sites: 8,192;

The v2 response uses an entity dictionary, ID relations, revision-bound opaque cursors, and `coverage.incompleteReason`. A scan-limited response is explicitly partial; a relation-page continuation is offered only when it can continue deterministically inside the captured scan window.
