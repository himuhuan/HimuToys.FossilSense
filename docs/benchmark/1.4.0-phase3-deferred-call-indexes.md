## Scope

This A/B validates deferring the seven call-fact secondary indexes during a full build. Explicit full-build databases and a first unpublished default database insert schema 15 facts without those B-trees, commit the complete generation, then create the indexes together and run `ANALYZE`. Ordinary dirty WAL writes retain all online indexes.

An existing default database is deliberately excluded until side-by-side generation-file publication is available; dropping indexes underneath active readers is not permitted.

The experiment is effective even when all deferred-index time is charged back to the write path. The remaining approximately 4.8 seconds above the full elapsed gate is not attributed to parser concurrency: parse took 8.564 seconds, while the combined SQLite fact/index path took 38.304 seconds. The next writer work is side-by-side publication and string-uniqueness bulk construction.
