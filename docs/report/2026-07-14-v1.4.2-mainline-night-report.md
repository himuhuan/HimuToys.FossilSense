# 2026-07-14 v1.4.2 主线夜间实施报告

## 结论

现已归档到 `docs/archive/plan/2026-07-13-v1.4.2-semantic-experience-strengthening.md` 的主线语义加强计划已经落地到 `release/v1.4.2`：五个用户可见方向不再各自解释函数或类型事实，而是建立在 schema 16、共享候选门面、同代 snapshot 与 all-open-document overlay 上。Rust、LSP、TypeScript、架构和发布门禁均通过，并生成了可安装的自包含 VSIX。

这份报告记录本次最终交付的全部范围。按收尾指示，本次只完成并提交 v1.4.2 主线，不再另开局部变量/宏扩展分支，也不生成第二份报告。

## 今晚完成的主线工作

### 1. schema 16 与 parser facts

- 将 parser fact contract 升到 v2、SQLite schema 升到 16；schema 15 只触发旁路全量重建，不保留生产双读。
- callable anchor 增加 canonical/presentation signature、arity shape、linkage、fidelity、精确范围与稳定 fingerprint。
- record 持久化 body/declaration/name 精确范围、range fidelity、revision 与声明区间 BLAKE3；alias 持久化 declarator shape、target fidelity、range、fingerprint 与声明区间 BLAKE3。
- 修复实现审查中发现的 canonical token 边界、default parameter AST、malformed alias、nested record wrapper、显式限定名空白规范化和跨文件 fingerprint 碰撞风险。

### 2. generation-pinned CandidateQueryService

- 新增请求期语义候选门面，在一个 pinned generation 上合并 durable exact-name rows 和工作区全部未同步 open-document overlays。
- dirty path 以 replacement/tombstone 语义遮蔽 base；lexical fallback 或 cancelled parse 明确使 coverage incomplete，禁止伪造唯一性。
- dirty include edges 在请求内刷新 ReachGraph；外部头根、开放原因与 generation 不混代。
- callable、record、alias 都通过 bounded exact-name recall 和有界 hydration；不在普通 completion 热路径读取磁盘或加载全库 catalog。
- 普通 completion 先走内存 prefix projection、再施加返回上限，避免大量无关 dirty symbol 把排序靠后的有效前缀候选提前截掉。

### 3. arity 与共享 callable presentation

- 统一 `CallSiteContext`、三态 arity compatibility（Compatible / Incompatible / Unknown）与候选 coverage。
- Hover、Definition、Signature Help、函数 completion documentation 和 Call Hierarchy 复用同一 callable group 与 presentation policy。
- complete/reliable arity 会排除 proven-incompatible 候选；partial/unsupported facts 保留可靠 fallback。Signature Help 选择首个 Compatible 作为 active signature，同时保持展示顺序稳定。
- 正式自由函数形态覆盖 direct name、explicit qualified name 与 parenthesized name；member/function pointer/callable object 等保持 unsupported evidence。

### 4. 严格 `.h/.c` counterpart

- counterpart graph 只接受 external linkage、exact canonical signature、closed source-to-header reach，并要求声明→定义和定义→声明两个方向 degree = 1。
- 删除旧 `ProjectKey`-only production pairing；ProjectKey 只保留为普通 completion evidence，不再建立绑定。
- paired Hover/Signature/completion documentation 采用 header representative；调用点 Definition 和 Call Hierarchy source-body-first；锚点 Definition 优先对侧。
- 修复了 1:N、N:1、共享 parser entity key、同 raw caller family、dirty include retarget、cancelled tombstone 与 relation expansion budget 的 Call Hierarchy 一致性问题。
- callable anchor 多名查询、候选 recall、dirty overlay call scan 与 relation materialization 都共享明确预算；一旦 recall 截断，严格 counterpart uniqueness 与 relation page 会同步标记 incomplete。
- 增加“未配对 callable anchor 仍返回自身”的 LSP 回归，避免声明被错误变成空跳转。

### 5. 完整 Record 与 typedef Hover

- record Hover 使用 revision-safe、byte/line-bounded `SourceExcerptReader` 读取精确声明，保留多行排版、注释和预处理行。
- reader 在读取前后校验同一文件句柄 metadata/identity、路径与精确声明区间 BLAKE3，防止原子替换、TOCTOU 或 size/mtime/file-id 都未变化的内容替换混入另一修订。
- typedef resolver 有界递归并检测 ambiguity、cycle、unsupported shape；只有当前 tier 收敛到唯一 record 时显示准确 `(aka. ...)` 和终点 record。C++ `using` alias trace 不在本次支持范围。
- record/alias 注释继续使用共享 Doxygen/XML renderer，并展示 path、tier、resolution confidence、reason、range fidelity 与降级原因。

### 6. 多根成员补全与一致性加固

- 每个 workspace root 都从同一 `DocumentRequestSnapshot` 建立自己的 pinned handle 与 dirty overlay；未索引但已打开的次根也能提供 record、alias 和 member facts。
- record/alias 解析可在根之间有界追踪，支持 alias 与终端 record 分处不同根；每一步都先按全局最高 scope tier 剪枝，低层 owner 不会在高层 owner 无匹配字段时被复活。
- dirty path 使用 replacement/tombstone 语义遮蔽 durable record、alias 与 member；空的 dirty record body 也会正确删除旧字段。
- resolved-owner 的链式类型读取与最终成员读取跨根共享 8192-row 预算，dirty overlay 优先；durable 与 overlay 的弱成员 fallback 也都使用稳定 prefix index、显式 scan cap 和 truncation 标记，不再无界扫描全库、巨型 record 或全部 dirty members。

## 审查中主动发现并修复的问题

- dirty overlay fact 不完整时，旧实现仍可能把 base 候选当成唯一；现在 completeness 会传遍 callable/alias/counterpart。
- 成员调用一度可能被 partial scanner 当成自由函数 fallback；现在保留 UnsupportedCallForm，显式限定/括号调用则进入正式形态。
- Definition 在无 counterpart 时过滤 origin，可能让独立声明返回空；现在保守保留锚点自身，并有 LSP 测试。
- completion documentation 一度按同名第一组取文档；现在先验证同代 symbol locator，再按 path、symbol/declaration range 精确选组。
- SourceExcerpt 的 size/mtime 校验存在读时替换窗口；现在增加同句柄前后 identity 校验与原子替换回归。
- 进一步发现 Windows 上内容可在 size、mtime、file-id 均不变时被原位替换；现在 record/typedef 持久化声明区间哈希，range-only reader 在返回前进行精确校验。
- Call Hierarchy 曾可能按 parser raw entity key 合并 1:N/N:1；现在直接复用共享 strict counterpart resolver，并为歧义 anchor 生成稳定 logical key。
- Call Hierarchy 的 multi-name SQL chunk、candidate recall 和 dirty overlay scan 曾各自有局部上限但缺少全局预算；现已统一限额，任何截断都会关闭唯一配对并向结果传播 incomplete。
- 多根成员补全曾可能复活低 tier owner、遗漏拆分在两个根的 alias/record、让 fallback 同名结果受 root 顺序影响，或先物化巨型 record 再截断；现已改为逐根同代查询、全局 tier/confidence 择优、全局 tier 剪枝、有界跨根 alias frontier 与 SQL/overlay 共享 `LIMIT+1` 预算。
- 普通 completion 曾在 prefix match 之前截断所有 dirty names，超过 2048 个无关项时可能漏掉后置有效候选；现在先做稳定前缀投影再限量，并加入规模回归。
- 发布门禁此前只能核对版本与文件名；现在 VSIX 内嵌 release-input manifest，记录源码输入、原生二进制、bundle、manifest、aggregate payload 和 source commit 的 SHA-256，hardening 会重新计算并拒绝陈旧包或源码漂移。
- clippy 发现的 needless lifetime 和格式漂移已清理；严格 `-D warnings` 通过。

## 保留的产品边界

FossilSense 仍不把候选伪装成编译器绑定。参数类型重载、模板、宏展开、成员调用、函数指针与 callable object 不在本次正式解析范围；include open/incomplete、alias 歧义/环/unsupported、source stale/oversized/unreadable 都会显式降级。Call Relations wire protocol 保持 v2，普通 completion 的 cost model 与 prefix-ranking 行为保持不变。
